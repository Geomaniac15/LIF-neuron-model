01 — OVERVIEW

Experimental Setup

The network stores 50 prototype memories — 5 per digit, each averaged from 10 training samples. Retrieval uses the softmax-energy update rule with β=4.0, operating on top-k=20 nearest memories per step. At test time, images are corrupted by randomly flipping 30% of pixels, then recovered via iterative convergence until the state norm change falls below tolerance.

The key departure from classical Hopfield networks is the energy function: the modern formulation replaces binary Hebbian weights with a continuous update that applies a softmax over dot-product similarities, giving exponential storage capacity and fast (often single-step) convergence to the nearest stored pattern.

The high pixel overlap (89.48%) with only 53.4% digit accuracy reveals the core behaviour: the network reliably converges to a crisp stored memory, but that memory is frequently the wrong digit — not a corrupted or partially-recovered state.

The models should be understood as an associative memory system, rather than a classifier. It selects the nearest stored pattern rather than verifying semantic correctness.

02 — ROBUSTNESS

Performance vs. Noise

The noise sweep tests retrieval accuracy across four noise levels, revealing a non-linear degradation curve with a sharp cliff between 30% and 50% noise. At low corruption, the initial state retains enough signal to bias the similarity computation toward the correct attractor basin. Beyond ~40% noise, the corrupted input sits roughly equidistant from multiple stored patterns and the network's choice becomes near-random.

Accuracy vs. Noise Level

The cliff at noise=0.5 is consistent with the theoretical observation that once noise exceeds the basin-of-attraction radius, retrieval fails catastrophically rather than gracefully. With only 5 prototypes per digit, the basins are relatively narrow. A denser memory set would shift this cliff rightward.

03 — PER-DIGIT ANALYSIS

Accuracy by Digit

Performance varies dramatically across digits. Digit 1 dominates at 90% accuracy, benefiting from its distinctive high-variance vertical stroke — highly separable in the 784-dimensional binary space. Digit 2 performs worst at 38%, owing to substantial visual overlap with the curved components of 1, 7, and 3.

05 — CONVERGENCE DYNAMICS

Trajectory Analysis

The trajectory images and convergence plots reveal a consistent pattern: the network commits to a memory within 1–2 iterations, then the similarity to that memory plateaus and holds for all remaining steps. This is characteristic of the modern Hopfield energy landscape — the softmax energy has very steep descent paths, so the update rule jumps to the bottom of the nearest attractor basin almost immediately.

The convergence plots show similarity to the best-matching stored memory jumping from ~10 to ~27 in the first step, while similarity to the true digit stays near 0 throughout. This means the network is not gradually recovering the correct digit — it locks onto the wrong attractor instantly and then stays there. Longer iterations don't help.

Approximate Convergence Step Distribution (Failure Cases Only, sampled from logs)

The distribution of convergence steps across failures skews toward 4–10 steps, with a long tail to ~20. Failures that converge in 4–5 steps tend to be confident wrong retrievals — strong pull toward an incorrect attractor. Longer convergence (>12 steps) often involves oscillation between two competing attractors before settling.

06 — DISCUSSION

Failure Modes and Next Steps

Three distinct failure regimes are visible in the data:

Basin collapse (noise < 0.4). The corrupted input still contains enough signal, but the prototype set doesn't accurately represent the within-class variance of that digit. The closest stored memory is structurally similar but belongs to a different digit — typically one with overlapping visual features.

Attractor competition (noise ~0.4–0.5). The initial state sits near the boundary of multiple basins. The top-k selection (k=20) draws from multiple digit classes simultaneously, and the weighted update can settle in a spurious mixed state that nearest-label decodes incorrectly.

Total corruption (noise > 0.5). The input has too little coherent structure. Similarity scores are roughly uniform across stored patterns. The network converges to an arbitrary attractor determined by noise — near-chance performance.

Failure Category Breakdown (Noise = 0.3)

Improvements to explore: Increasing n_prototypes (from 5 to 20–50 per digit) would reduce basin collapse by better covering within-class variance. Tuning β — the softmax inverse temperature — controls the sharpness of the update; higher β commits faster to the nearest memory but narrows basin radii. A hierarchical retrieval strategy (coarse class selection first, then fine-grained prototype matching) could help disambiguate visually similar digits like 2/7 and 9/4.
